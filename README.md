# health-widget

Приватный always-on-top виджет показателей здоровья для Wayland-линукса.
Читает метрики из JSON, показывает их поверх всех окон и **скрывается от шаринга экрана**.

Целевая среда — **KDE Plasma 6.6+ (KWin/Wayland)**, где виджет можно оставить видимым
локально и при этом исключить из захвата. GNOME/Mutter поддержан как запасной вариант
(там доступно только авто-скрытие).

## Как достигается приватность (важно прочитать)

На Windows/macOS есть системный флаг «не захватывать это окно». На Wayland клиент сам такого
флага выставить не может. Что доступно по средам:

0. **KWin `excludeFromCapture` — настоящее исключение из захвата (только KDE Plasma 6.6+).**
   У KWin с 6.6 есть свойство окна «Hide from Screencast»: окно видно тебе, но **не попадает
   в поток** записи/шаринга (в т.ч. «Весь экран», Zoom/Meet/OBS через портал). Оконных правил
   для него в 6.6 ещё нет (будут в 6.7), поэтому мы ставим свойство маленьким KWin-скриптом
   `health-widget-exclude` — он автоматически помечает окно виджета при каждом запуске.
   Это самый надёжный слой; при нём авто-скрытие не нужно (`HEALTH_AUTO_HIDE=0`).
   Ставится через `./install-kde-privacy.sh` (или автоматически в `setup.sh` на KDE).
1. **Шарь одно окно, а не весь экран.** При «оконном» шаринге в поток попадает только выбранное
   окно; виджет — отдельное окно и туда не входит. Работает на любом композиторе.
2. **Авто-скрытие при обнаружении захвата** (`HEALTH_AUTO_HIDE=1`, дефолт; выключай на KDE).
   Фоновый поток детектит активный захват **кросс-десктопно через PipeWire** (`pw-dump`:
   нода композитора `Stream/Output/Video` в состоянии `running`); на GNOME есть резервный путь
   через `org.gnome.Mutter.ScreenCast`. Как только начинается захват — виджет прячется сам
   (best-effort). В отличие от слоя 0, тут виджет пропадает и у тебя.
3. **Ручной хоткей** (SIGUSR1) — мгновенно спрятать/показать в любой момент.

> ⚠️ На KDE 6.6+ слой 0 даёт настоящую приватность даже при шаринге всего экрана. На GNOME
> гарантии на уровне ОС нет — там работает связка «шарь одно окно» + авто-скрытие + хоткей.
> Перед боевым созвоном всё равно **проверь сам** (см. «Проверка»).

## Установка

```bash
cd ~/projects/health-widget
./setup.sh            # поставит apt-зависимости, Rust (если нет), соберёт релиз
```

Или вручную:

```bash
sudo apt-get install -y build-essential pkg-config \
  libwayland-dev libxkbcommon-dev libx11-dev libxcursor-dev libxrandr-dev libxi-dev libgl1-mesa-dev
# Rust: https://rustup.rs
cargo build --release
```

## Запуск

**KDE Plasma (рекомендуется):** авто-скрытие не нужно, приватность даёт KWin.

```bash
HEALTH_AUTO_HIDE=0 ./target/release/health-widget
```

Перед первым запуском один раз поставь KWin-исключение (или это сделает `setup.sh` на KDE):

```bash
./install-kde-privacy.sh
```

**GNOME / прочее:** полагаемся на авто-скрытие (включено по умолчанию):

```bash
./target/release/health-widget
```

Данные по умолчанию: `~/.config/health-widget/metrics.json`.

## Формат JSON

Структурированный:

```json
{
  "title": "Здоровье",
  "metrics": [
    { "label": "Пульс", "value": "62 bpm" },
    { "label": "Сон",   "value": "7ч 10м" }
  ]
}
```

Либо плоский объект (ключи выводятся по алфавиту):

```json
{ "пульс": 62, "сон": "7ч 10м", "шаги": 8420 }
```

Файл перечитывается автоматически при изменении — можешь писать в него из любого своего скрипта.

## Хоткей скрыть/показать (SIGUSR1)

Wayland не даёт приложению самому перехватить глобальную клавишу, поэтому используем сигнал.
Команда для ярлыка: `pkill -SIGUSR1 -x health-widget`.

**KDE Plasma:** System Settings → Keyboard → Shortcuts → Custom Shortcuts → добавь команду
`pkill -SIGUSR1 -x health-widget` и назначь клавишу (например `Meta+H`).

**GNOME:** Settings → Keyboard → View and Customize Shortcuts → Custom Shortcuts → `+`,
Command: `pkill -SIGUSR1 -x health-widget`, клавиша например `Super+H`.

Теперь клавиша мгновенно прячет/показывает виджет.

## Онлайн-транскрипция под осциллографами

Под каждым включённым каналом (🎤 микрофон, 🔊 звук программы/созвона) в реальном времени
идёт транскрипция того, что попадает в захват. Распознавание **потоковое и полностью офлайн**
(Vosk): текст достраивается по мере речи — накопленные фразы обычным цветом, текущая гипотеза
серым курсивом. Аудио никуда не отправляется, что важно для приватного виджета.

Как это устроено: канал уже пишет f32 PCM для волны; те же сэмплы ресемплятся в 16 кГц и
скармливаются локальному Python-хелперу `scripts/vosk_stream.py` (пакет `vosk` в отдельном
venv). Rust-либы/линковка libvosk не нужны — тем же приёмом, что и `pw-record`/`pw-dump`.

Один раз поставь движок и русскую модель (~46 МБ, ставится в `~/.local/share/health-widget`):

```bash
./install-transcribe.sh
```

Без установки виджет работает как раньше — просто без текста под осциллографами.
Выключить транскрипцию, не удаляя модель: `HEALTH_TRANSCRIBE=0`.

## Настройка (переменные окружения)

| Переменная              | Смысл                               | Дефолт |
|-------------------------|-------------------------------------|--------|
| `HEALTH_JSON`           | путь к JSON                          | `~/.config/health-widget/metrics.json` |
| `HEALTH_X` / `HEALTH_Y` | позиция окна (px)                    | 40 / 60 |
| `HEALTH_W` / `HEALTH_H` | размер окна (px)                     | 260 / 200 |
| `HEALTH_BG_ALPHA`       | непрозрачность подложки 0..255       | 220 |
| `HEALTH_CLICK_THROUGH`  | клики сквозь окно (1/0)              | 0 |
| `HEALTH_AUTO_HIDE`      | авто-скрытие при захвате (1/0)       | 1 |
| `HEALTH_TRANSCRIBE`     | онлайн-транскрипция (1/0)            | 1 |
| `VOSK_MODEL`            | каталог модели Vosk                 | `~/.local/share/health-widget/vosk-model-small-ru-0.22` |
| `VOSK_PYTHON`           | python с пакетом `vosk`             | `~/.local/share/health-widget/venv/bin/python` |

Пример:

```bash
HEALTH_X=3000 HEALTH_Y=40 HEALTH_CLICK_THROUGH=1 ./target/release/health-widget
```

Диагностика детектора захвата (полезно проверить свой инструмент созвона):

```bash
./target/release/health-widget --check-capture   # exit 0 = захват идёт, 1 = нет
```

## Проверка приватности (сделай до боевого созвона)

**KDE Plasma 6.6+ (слой 0, excludeFromCapture):**

1. Запусти виджет (`HEALTH_AUTO_HIDE=0 …`) — он виден у тебя.
2. Начни шаринг «Весь экран» в Zoom/Meet или запись экрана Spectacle.
3. В потоке/записи виджета быть **не должно**, при этом у тебя на экране он остаётся.
   (Если виден — проверь, что стоит KWin-скрипт: `./install-kde-privacy.sh`.)

**GNOME (авто-скрытие):**

1. Запусти виджет.
2. Открой запись экрана (`Super+Alt+R`) или тестовый шаринг «Весь экран».
3. Виджет должен исчезнуть в течение ~1 сек. Останови запись — вернётся.
4. Отдельно проверь «шаринг одного окна»: расшарь браузер/IDE — виджета в записи быть не должно.

Если авто-скрытие не срабатывает у твоего инструмента — полагайся на «шарь одно окно» + хоткей.

## Автозапуск

На KDE `./install-kde-privacy.sh` уже создаёт `~/.config/autostart/health-widget.desktop`
с `HEALTH_AUTO_HIDE=0`. Вручную (или на GNOME) положи туда:

```ini
[Desktop Entry]
Type=Application
Name=Health Widget
Exec=env HEALTH_AUTO_HIDE=0 /home/mgu/projects/health-widget/target/release/health-widget
X-GNOME-Autostart-enabled=true
```

На GNOME убери `HEALTH_AUTO_HIDE=0`, если хочешь оставить авто-скрытие.
