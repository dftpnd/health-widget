# health-widget — правила для агента

## Комментарии в коде — не писать

Никаких комментариев в исходниках. Ни в Rust (`//`, `///`, `//!`, `/* */`),
ни в Python (`#`). Код объясняет себя сам: понятные имена, маленькие функции,
явные типы. Если тянет написать комментарий — это сигнал переименовать/разбить,
а не пояснять прозой.

- Не добавляй комментарии в новый код.
- Не возвращай удалённые комментарии при правках существующего кода.
- Docstring'и модулей/функций в Python — тоже не добавлять.
- Исключение только для функционально обязательного: шебанг (`#!/usr/bin/env …`),
  `#`-директивы в файлах-данных (`it_hotwords.txt`, `it_corrections.tsv`), которые
  парсер трактует как разделители. Это не пояснения, а часть формата.

## Прочее

- Комментарии/строки UI — на русском (в тон проекту), если уж строка нужна.
- Ветка одна — `master`. Не создавать git-ветки без явной просьбы.
- Идиома проекта: шеллить готовые CLI (`wl-copy`, `pw-record`, `qdbus6`, …),
  а не тянуть новые зависимости, где это дёшево.
