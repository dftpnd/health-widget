# Гигиена запуска/рестарта health-widget

Дата: 2026-07-10. Статус: утверждено.

## Проблема

Каждый канал звука поднимает свой whisper-процесс с копией модели large-v3
(~4 ГиБ VRAM). Рестарт виджета перегружает модель (~20–30 с), при быстрых
рестартах старый python ещё выгружает CUDA, пока новый уже грузит модель —
транзиентное пересечение по VRAM. 2026-07-09 22:23 серия из четырёх рестартов
за 9 минут закончилась исчерпанием NVKMS-пула драйвера NVIDIA
(`Failed to allocate NVKMS memory for GEM object`) и фризом курсора: KWin не мог
выделить буфер курсорной плоскости.

## Решение (утверждено: «дети + гигиена», без демона)

### 1. Single-instance замок — новый `src/instance.rs`

- Lock-файл `$XDG_RUNTIME_DIR/health-widget.lock` (tmpfs, чистится при ребуте),
  содержимое — PID держателя.
- При старте `flock(LOCK_EX | LOCK_NB)`. Если занят: прочитать PID старого →
  `SIGTERM` → поллинг flock до ~15 с → не освободился → `SIGKILL` → взять лок.
- flock отпускается ядром при смерти держателя — stale-локов не бывает.
- fd лока живёт всё время работы процесса.

### 2. Ожидание смерти whisper до спавна нового

- После взятия лока: пока `pgrep -f <data_dir>/whisper_stream.py` непуст —
  ждать до ~10 с, затем `pkill -9 -f` остатки.
- Матч по полному пути скрипта, чтобы не задеть чужие процессы.
- Гарантия: на GPU не больше одной модели на канал в любой момент.

### 3. Graceful shutdown

- `sigaction` на SIGTERM/SIGINT (без новых зависимостей, libc уже есть) →
  AtomicBool → в `update()` по флагу `ViewportCommand::Close` → eframe
  завершается штатно → существующие Drop'ы убивают whisper и pw-record.
- В `Transcriber::drop` после `kill()` добавить `wait()` — не оставлять зомби.
- Виджет перерисовывается не реже раза в 500 мс, значит сигнал заметится
  максимум через полсекунды.

### 4. VRAM-клапан перед спавном whisper

- Перед спавном в `Transcriber::start`: `nvidia-smi --query-gpu=memory.free`.
- Свободно меньше порога (default 5000 MiB, env `HEALTH_STT_MIN_FREE_MIB`) →
  телеметрия `stt.low_vram`, вернуть `None`: виджет живёт без транскрипции,
  композитор не доводится до NVKMS-фриза.
- `nvidia-smi` недоступен/непарсибелен → клапан пропускает (fail-open).

## Не делаем (YAGNI)

Демон whisper, выгрузка модели по idle, смена stdin/stdout-протокола,
systemd-рестарты, RSS-утечка виджета, 4-секундный KWin-опрос позиции.

## Проверка

- Юнит-тесты: парсинг вывода nvidia-smi и порога.
- Вживую: два запуска подряд (старый умирает, python ровно один);
  `kill -TERM` (дети умирают, зомби нет); старт при забитой VRAM
  (STT не стартует, виджет живёт).
