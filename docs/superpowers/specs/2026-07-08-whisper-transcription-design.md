# Замена Vosk на faster-whisper для транскрипции

Дата: 2026-07-08

## Проблема

Текущая транскрипция (Vosk `vosk-model-small-ru-0.22`, стриминг через Python-хелпер
`scripts/vosk_stream.py`) плохо распознаёт речь, особенно IT-англицизмы в русской речи.
Нужно поднять качество; небольшим лагом по скорости можно пожертвовать. Основной язык — русский.

## Решение (кратко)

Заменить движок распознавания на **faster-whisper** (модель Whisper large-v3 на движке
CTranslate2), работающий на GPU (RTX 5080, 16 ГБ). Тот же архитектурный приём, что у Vosk:
не линкуем библиотеку в бинарь, а шеллим Python-хелпер. Whisper заметно точнее на смешанной
русско-английской речи и поддерживает два рычага для IT-терминов: `hotwords`/`initial_prompt`
(подсказка при распознавании) и пользовательский словарь пост-коррекции.

Whisper работает не по-словно, а фразами: под осциллографом текст появляется целиком после
паузы в речи (лаг ~1–3 с). Это осознанный выбор — простота и максимум качества.

## Окружение (проверено)

- GPU: NVIDIA RTX 5080, 16 ГБ VRAM, Blackwell/sm_120, драйвер 595.71.05 (умеет CUDA 13).
- Python 3.12.13 доступен (`~/.local/bin/python3.12`); `uv` 0.11.26 установлен.
- Системного CUDA toolkit / cuDNN нет — CUDA-либы ставим pip-колёсами в venv
  (`nvidia-cublas-cu12`, `nvidia-cudnn-cu12`).
- Существующий venv транскрипции на Python 3.14 — под него колёс ctranslate2 нет,
  поэтому whisper ставим в **отдельный venv на Python 3.12**.

## Архитектура: что меняется, что остаётся

Rust-сторона почти не меняется. `Feeder` (`src/transcribe.rs:61`) уже ресемплит аудио в
моно s16le @16000 — это ровно вход Whisper. Reader-поток (`src/transcribe.rs:122-153`) уже
разбирает построчный JSON `{"final": …}` и пишет финальные сегменты в БД.

Меняется:

- **Python-хелпер**: `scripts/vosk_stream.py` → новый `scripts/whisper_stream.py`
  (VAD-нарезка на фразы + инференс faster-whisper + hotwords/prompt + пост-коррекция).
- **Partials убираются**: Whisper их не выдаёт. Серый «на лету» текст исчезает; под
  осциллографом появляется законченная фраза. UI показывает пустой `partial`.
- **`src/transcribe.rs`**: пути к питону/скрипту/модели указывают на whisper-стек;
  комментарии и имена хелперов обновить. Константа `STT_RATE = 16000` остаётся.

Каналов по-прежнему два (микрофон `🎤 я` + системный звук `🔊 телемост`), каждый спавнит
свой хелпер → две копии модели в VRAM (~3–4 ГБ суммарно из 16 ГБ — помещается).

## Компонент: `scripts/whisper_stream.py`

Тот же контракт stdin/stdout, что и `vosk_stream.py`: читает сырой s16le@16000 из stdin,
печатает построчный JSON в stdout. Отличия внутри:

```
model = WhisperModel(WHISPER_MODEL, device=WHISPER_DEVICE, compute_type=WHISPER_COMPUTE)
loop:
    копим PCM из stdin
    Silero-VAD (vad_filter=True) ищет конец фразы (пауза ~0.5 с)
    segment → model.transcribe(segment, language="ru",
                               hotwords=<топ IT-термины из it_hotwords.txt>,
                               initial_prompt=<та же подсказка>)
    text → пост-коррекция по it_corrections.tsv
    print {"final": text}  (flush)
```

- Печатает только `{"final": …}`; `{"partial": …}` не шлёт.
- `language="ru"` фиксирован. Англицизмы Whisper вставляет латиницей внутри русской речи.
- VAD встроен в faster-whisper — отдельная библиотека не нужна.
- Логи движка глушим (только JSON в stdout), как в vosk_stream.py.
- Аргумент argv[1] — путь к модели (или её имя для авто-скачивания), как сейчас.

## Компонент: база IT-англицизмов (оба рычага)

Два текстовых файла в data-dir (`~/.local/share/health-widget/`), редактируются вручную.
Оба опциональны: нет файла → рычаг не применяется.

- **`it_hotwords.txt`** — один термин на строку, топ-частотные
  (`Kubernetes`, `Docker`, `pull request`, `deploy`, `backend`, …). Хелпер грузит первые
  ~80 строк в `hotwords`/`initial_prompt` (ограничение ~224 токена подсказки Whisper).
  Рычаг №1 — склоняет распознавание к правильным терминам.

- **`it_corrections.tsv`** — строки `ослышка<TAB>правильно`, например:
  ```
  кубернетис	Kubernetes
  пул реквест	pull request
  дэплой	deploy
  ```
  Рычаг №2 — замены на готовом тексте по границам слов, регистронезависимо.
  Размер не ограничен.

В комплекте — стартовый набор ~40 терминов; дальше пользователь наполняет сам.

## Установка (`install-transcribe.sh` переписывается)

```
uv venv ~/.local/share/health-widget/venv-whisper --python 3.12
uv pip install --python <venv-whisper>/bin/python \
    faster-whisper nvidia-cublas-cu12 nvidia-cudnn-cu12
# модель large-v3 скачивается при первом запуске (кэш HuggingFace) либо предзагружается
```

Старый `venv/` (Vosk) и Vosk-модель на время миграции не трогаем; после успешной проверки
их можно удалить.

## Гейт проверки (до правки Rust)

Прежде чем менять Rust — отдельный smoke-тест хелпера: скормить `whisper_stream.py` готовый
WAV (например дорожку из `calls/<id>/mic.wav`) через stdin и убедиться, что:

1. модель грузится на GPU без ошибки на sm_120 (Blackwell);
2. инференс реально идёт на GPU (видно в `nvidia-smi`);
3. на выходе осмысленный русский текст с латинскими англицизмами.

Если ctranslate2 не поддержит Blackwell — чиним здесь (другая версия ctranslate2 / сборка /
`compute_type="int8_float16"`), пока Rust не тронут. Это блокирующий гейт: без зелёного
smoke-теста интеграцию в Rust не начинаем.

## Конфигурация и фолбэк

Переменные окружения (переопределяют дефолты):

- `WHISPER_PYTHON` — интерпретатор (дефолт `<data-dir>/venv-whisper/bin/python`).
- `WHISPER_MODEL` — модель (дефолт `large-v3`).
- `WHISPER_DEVICE` — устройство (дефолт `cuda`).
- `WHISPER_COMPUTE` — тип вычислений (дефолт `float16`; `int8_float16` для экономии VRAM).
- `HEALTH_TRANSCRIBE=0` — выключает транскрипцию целиком (как сейчас).

Нет venv/модели/скрипта → `Transcriber::start()` возвращает `None`, канал работает без
текста (текущее поведение при отсутствии окружения).

## Что НЕ делаем (YAGNI)

- Не строим общий сервер на два канала — две копии модели в 16 ГБ помещаются, простота важнее.
- Не делаем UI для редактирования базы терминов — файлы правятся руками.
- Не оставляем гибрид Vosk+Whisper — чистая замена.

## Не меняется

- Захват звука (`src/audio.rs`, `pw-record` f32@44100, два канала, фан-аут в осциллограф /
  STT / WAV).
- Схема БД (`transcript` / `calls` / `tracks`) и запись финалов через `TranscriptLog::append`.
- CLI-флаги `--transcript`, `--calls`, `--export`.
- Запись WAV-дорожек колов.

Итог для пользователя: текст под осциллографом и в БД становится точнее (особенно IT-англицизмы),
ценой лага ~1–3 с и появления текста фразами вместо по-словного.
