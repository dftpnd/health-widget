# Виртуальная камера с аватаром-пандой — дизайн

Дата: 2026-07-09
Статус: одобрен к планированию

## Цель

Виртуальная веб-камера (`/dev/videoN` через `v4l2loopback`), которую видят обычные
приложения (Zoom, Meet, Discord, OBS, браузер). В кадре — статичный SVG-аватар
(панда, `/home/mgu/Pictures/panda.svg`), а на месте рта — живой осциллограф,
рисуемый из свежих сэмплов микрофона: говоришь — «рот» шевелится. Управление
из виджета: тумблер вкл/выкл.

## Не-цели (YAGNI)

- Нет фонемного липсинка, морганий, покачиваний, эмоций. Только рот-осциллограф.
- Нет редактора аватара/риггинга. Панда статична, рот — одна коробка-осциллограф.
- Нет юнит-тестов на v4l2 (это I/O к ядру); проверка — поведением.
- Виджет не делает `modprobe` (root); загрузка модуля — ручной шаг/инструкция.

## Решения (зафиксировано на брейншторме)

- Настоящая вирткамера через **`v4l2loopback`** (ядрёный модуль, `sudo modprobe`).
- SVG растеризуем **чистым Rust-крейтом `resvg`** (без root, без системных пакетов),
  один раз на старте — панда статична.
- Кадры пишем **сырьём прямо в `/dev/videoN`**, без `ffmpeg`. Формат камере задаём
  через `libc::ioctl(VIDIOC_S_FMT)`.
- Формат пикселей — **YUYV (YUV 4:2:2)** ради максимальной совместимости потребителей.
- Аудио берём из **default source (микрофон)** — рот шевелится, когда говоришь ты.

## Стек и зависимости

- Новая зависимость: `resvg` (+ транзитивно `usvg`/`tiny-skia`). Больше ничего.
- Переиспользуем: `AudioMonitor` (`src/audio.rs`) — уже пишет моно-`f32` @44100
  через `pw-record` в кольцо `samples: Arc<Mutex<VecDeque<f32>>>` (CAP=2048) и отдаёт
  его через `snapshot(&mut Vec<f32>)`. Аватар дёргает `snapshot` каждый кадр.
- `libc` (уже есть) — для `ioctl`/`write` к устройству.
- `telemetry` (уже есть, JSONL) — для логирования серий ошибок записи.

## Компоненты

Новый модуль **`src/avatar.rs`** — вся логика инкапсулирована здесь, наружу торчит
маленький контроллер.

```
pub struct Avatar { … }               // хэндл: держит поток, atomic-флаг, путь устройства
impl Avatar {
    pub fn start(cfg: &AvatarCfg, audio: Arc<AudioMonitor>) -> Result<Avatar, AvatarError>
    pub fn stop(self)                  // atomic-флаг + join
    pub fn is_running(&self) -> bool
    pub fn last_frame(&self) -> Option<ColorImage>   // для превью в виджете
}
impl Drop for Avatar { … }            // stop при дропе
```

`start`:
1. Открыть `/dev/videoN`, задать формат `set_format(fd, w, h)` → YUYV, W×H, fps.
   Единственный изолированный островок unsafe FFI (`libc::ioctl`, `VIDIOC_S_FMT`,
   структура `v4l2_format`). Обёртка над `RawFd` закрывает fd по RAII.
2. Растеризовать `panda.svg` через `resvg` → кэш `Vec<u8>` RGBA. Фон тёмный
   (как на текущем скрине), панда вписана по аспекту (viewBox 359.458×265).
3. Заспавнить поток-компоновщик.

## Поток данных (30 fps, цикл компоновщика)

```
mic (default source)
  └─ AudioMonitor (pw-record, уже есть) ─ кольцо f32
        └─ avatar-thread: snapshot(&mut buf)
              ├─ клонирует кэш панды (RGBA)
              ├─ рисует осциллограф из buf в коробку рта (полилиния, scope_color)
              ├─ RGBA → YUYV
              ├─ write() кадр в /dev/videoN
              └─ (по флагу) кладёт RGBA в last_frame для превью
        спит до следующего кадра (~33 мс при 30 fps)
```

Если основной `AudioMonitor` виджета уже слушает микрофон — шарим один `Arc`,
второй захват не плодим.

## Конфиг

Новая под-структура `avatar` в существующем `Config` (в духе уже имеющихся
audio/autopilot/shot), все поля с дефолтами:

```
avatar: {
    svg_path:    "/home/mgu/Pictures/panda.svg",
    device:      "/dev/video10",
    width:       640,
    height:      480,
    fps:         30,
    mouth:       { x: 150, y: 330, w: 340, h: 90 },   // подгоняется по превью
    scope_color: [220, 30, 20],                        // красный смайл
    scope_gain:  6.0,                                  // усиление амплитуды
}
```

Коробка рта — прямоугольник в координатах **выходного кадра** (пиксели).
Правится в конфиге, перезапуск камеры подхватывает.

## Управление в виджете

Тумблер **«🐼 Камера»** (вкл/выкл) рядом с существующими контролами. Состояние —
новая под-структура `App.avatar`: `Option<Avatar>`, признак running, последняя ошибка.

- Вкл: проверяем наличие `/dev/video10` → `Avatar::start` → индикатор «в эфире».
- Устройства нет → кнопка неактивна + подсказка с командой `modprobe`.
- Ошибка старта (SVG/формат) → текст ошибки в статус, виджет не падает.

**Превью (встроенное).** Раз RGBA-кадр и так в руках — маленькая панель показывает
текущий кадр как egui-текстуру (по флагу `last_frame`). Двойная польза: видно, что
камера рисует, без сторонних приложений; по нему же ловим коробку рта.

## Обработка ошибок

Единый `enum AvatarError`, наружу — текст в статус; виджет не падает.

| Ситуация | Поведение |
|---|---|
| Нет `/dev/videoN` | Кнопка неактивна + подсказка `sudo modprobe v4l2loopback video_nr=10 card_label="Panda" exclusive_caps=1` |
| `ioctl(S_FMT)` отвалился | Ошибка в статус, поток не стартует |
| SVG не читается/не парсится | Ошибка в статус |
| `write()` → EAGAIN / потребитель отвалился | Пропускаем кадр, продолжаем; серию ошибок логируем в telemetry |
| Мик недоступен | Осциллограф — плоская линия (рот «молчит»), камера живёт |

## Жизненный цикл потока

- Стоп: `AtomicBool` + `JoinHandle`. `stop()` ставит флаг, поток дорисовывает кадр
  и выходит, затем `join()`. Никаких detached-потоков.
- `impl Drop for Avatar` вызывает `stop`.
- fd закрывается при выходе потока (RAII-обёртка над `RawFd`).

## Персистентность модуля (вне кода, одноразово)

Чтобы `/dev/video10` жил после ребута — drop-in в `/etc/modules-load.d/` и
`/etc/modprobe.d/` с опциями (`video_nr=10 card_label="Panda" exclusive_caps=1`).
Это инструкция для пользователя (root), не задача виджета.

## Верификация (end-to-end, по порядку)

1. `sudo modprobe v4l2loopback video_nr=10 card_label="Panda" exclusive_caps=1`
   → появляется `/dev/video10`.
2. Собрать, включить тумблер → **встроенное превью** показывает панду; говоришь в
   мик — осциллограф во рту шевелится. Первичная проверка, без сторонних тулов.
3. Внешний потребитель: тест вебки в браузере (`getUserMedia`) или KDE-камера →
   выбрать «Panda» → тот же кадр. Подтверждает, что устройство видно приложениям.
4. Коробку рта/усиление подгоняем по превью, правим конфиг.
