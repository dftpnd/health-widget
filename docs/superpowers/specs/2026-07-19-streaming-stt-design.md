# Стриминговая транскрипция с LocalAgreement-2

Дата: 2026-07-19. Область: только `scripts/whisper_stream.py` (+ его тесты). Rust-сторона
(`src/transcribe.rs`, `src/audio.rs`, webmic/оверлей) не меняется — она уже принимает и
`{"partial": ...}`, и `{"final": ...}`.

## Проблема

Сейчас скрипт — чистый endpointing по тишине: фраза копится, пока RMS не упадёт ниже
порога на 0.6 с, затем декодируется целиком. Пока человек говорит, наружу не выходит
ничего (до 15 с при монологе), плюс хвост тишины, плюс декод всей фразы, плюс для
коротких фраз второй стабилизационный декод. Итоговый лаг — от ~1.5 с до десятков секунд.

## Цель

- Черновой текст (`partial`) появляется через ~1 с после произнесения.
- Финальный текст течёт во время речи, не дожидаясь пауз: ~1.5–2.5 с после произнесения.
- Финал после замолкания: ~0.4 с тишины + один декод.
- Антигаллюцинационные свойства не хуже текущих.

## Архитектура

Главный цикл переписывается со схемы «копим фразу → декод по тишине» на скользящий
буфер с коммитом по согласию двух последовательных декодов (LocalAgreement-2, схема
whisper_streaming).

### 1. Буфер и темп декода

Поток-читатель stdin (`_drain_stdin`) остаётся. Главный цикл копит s16le-фреймы в
скользящий аудиобуфер с оффсетом (в сэмплах от начала потока). Энергетический гейт
(текущий RMS по фреймам 20 мс) различает речь/тишину. Декод запускается, когда с начала
прошлого декода накопилось ≥ 1.0 с нового аудио и в буфере есть речь. Декод медленнее
реального времени сам замедляет темп — следующий прогон забирает всё накопившееся.
Чистую тишину не декодируем.

### 2. Декод

`model.transcribe(buffer, language="ru", beam_size=5, vad_filter=True,
condition_on_previous_text=False, hotwords=..., word_timestamps=True)`.
Сегменты прогоняются через существующий `is_hallucination` (текст + no_speech_prob +
avg_logprob + compression_ratio); отбракованные выпадают до разбора на слова. Оставшиеся
сегменты разворачиваются в плоский список слов `(text, start, end)`.

### 3. Коммит (LocalAgreement-2)

Список слов текущего декода сравнивается со списком прошлого декода позиционно, по
нормализованной форме слова (lower, без пунктуации — как `_match_form`). Максимальный
совпавший префикс сверх уже закоммиченного объявляется финальным и не переделывается.
Для речи, пережившей ≥2 декодов, это заменяет прошлый гейт «двойной декод с
пертурбацией»: нестабильный декод шума не совпадает между прогонами и не коммитится.
`texts_agree`, `_match_form`, `STABILITY_MIN_SIMILARITY` и `MAX_SEGMENT` удаляются;
пертурбация (`PERTURB_*`, `STABILITY_MAX_SECONDS`) сохраняется только для флаша
коротких фраз (см. §5).

### 4. Вывод

- После каждого декода незакоммиченный хвост → `{"partial": "..."}` (одной строкой,
  без коррекций).
- Закоммиченные слова копятся в отложенный финал; он уходит как `{"final": ...}` с
  `apply_corrections` на границе предложения (`.`, `?`, `!`, `…` в конце слова), при
  флаше по тишине или по достижении 30 слов.

### 5. Флаш по тишине

Тишина ≥ 0.4 с (`SILENCE_TAIL` 0.6 → 0.4) при непустой речи в буфере: финальный декод
буфера, весь его текст коммитится (пауза — надёжная граница, как в текущей схеме),
отложенный финал уходит, буфер очищается, печатается пустой `partial`. Речь короче
`MIN_SPEECH` (0.3 с) по-прежнему отбрасывается без декода.

Если к моменту флаша ничего не закоммичено и буфер ≤ 3 с (`STABILITY_MAX_SECONDS`) —
изолированная короткая фраза, LocalAgreement не успел сработать — делается второй
декод пертурбированного аудио (`PERTURB_*`) и коммитится только согласный префикс
двух прогонов: защита от галлюцинаций на коротких шумовых всплесках сохраняется.

### 6. Обрезка буфера

После коммита, если буфер длиннее 12 с (`MAX_BUFFER`), аудио срезается до конца
последнего закоммиченного слова (оффсет сохраняется, таймстемпы согласованы).
Патология «буфер растёт, а коммитов нет» — принудительная обрезка до последних 12 с
со сбросом состояния согласия (прошлый список слов забывается).

### EOF

По закрытию stdin — как флаш по тишине: финальный декод остатка, коммит, выход.

## Ошибки и граничные случаи

- Галлюцинация, стабильная в двух прогонах: ловится сегментным `is_hallucination`
  (чёрный список, метрики) — контур сохраняется целиком.
- Коррекция-фраза, разорванная границей двух финалов, не применится — тот же риск,
  что сегодня на границе двух фраз; принимаем.
- Пунктуация/регистр на стыках коммитов могут отличаться от офлайн-декода; принимаем
  как цену стриминга.

## Тесты

В `scripts/test_whisper_stream.py`, чистые функции без модели:

- нормализация слова и согласование префиксов (включая: пустые списки, полный сдвиг
  гипотезы, частичное совпадение, повторные слова);
- правила флаша финала (граница предложения, лимит слов);
- арифметика обрезки буфера (оффсеты, срез по таймстемпу слова).

Существующие тесты `is_hallucination`, hotwords, corrections не трогаются; класс
`TestTextsAgree` удаляется вместе с `texts_agree`.
