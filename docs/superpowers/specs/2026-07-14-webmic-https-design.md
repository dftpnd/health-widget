# Веб-микрофон наружу: https на 0.0.0.0 с токеном вместо localhost-туннеля

## Зачем

Второй комп теперь не в локалке, а в интернете; у основного компа есть публичный IP.
ssh-туннель на каждое подключение неудобен. Хотим: открыть ссылку в браузере второго
компа — и говорить. Ограничение браузера: `getUserMedia` работает только в secure
context, значит наружу — только https.

## Как пользоваться

1. Разово: на роутере пробросить TCP 8787 на основной комп.
2. Кнопка 🌐 включает сервер и копирует в буфер готовую ссылку
   `https://<ip>:8787/?t=<токен>`.
3. На втором компе открыть ссылку, один раз принять предупреждение о
   самоподписанном сертификате — дальше как раньше: «Говорить» → текст на
   странице и в окне `hw-web`.

## Архитектура

Режим один — https; отдельного localhost-http-режима не остаётся, ssh-туннель
больше не нужен.

- **Сервер** — `tiny_http::Server::https` на `0.0.0.0:8787`, фича `ssl-openssl`
  (системный openssl). Остальное (поток, `recv_timeout`-цикл, статика,
  `/api/audio`, авто-стоп STT) — без изменений.
- **Сертификат и ключ** — `~/.local/share/health-widget/webmic-cert.pem` /
  `webmic-key.pem`. Если файлов нет — генерятся при включении кнопки шеллом:
  `openssl req -x509 -newkey rsa:2048 -nodes -days 3650 -subj /CN=health-widget`.
  Файлы переживают рестарты — предупреждение браузер показывает один раз
  на устройство.
- **Токен** — `~/.local/share/health-widget/webmic-token`, генерится
  `openssl rand -hex 16`, если файла нет. Каждый запрос (страница, статика,
  `/api/audio`) обязан нести `?t=<токен>`, иначе `403` без тела. Сравнение —
  точное совпадение строки.
- **Страница** — `web/src/App.jsx` читает `t` из `location.search` и добавляет
  его к POST `/api/audio`. Единственная правка фронта; пересборка `npm run build`.
- **Ссылка** — `connect_hint()` возвращает `https://<ip>:8787/?t=<токен>`.
  IP: `HEALTH_WEBMIC_HOST`, если задан → публичный через
  `curl -s --max-time 2 api.ipify.org` → LAN-IP (`hostname -I`) как фолбэк.

## Решённые развилки

- TLS встроенный самоподписанный, не реверс-прокси с настоящим сертификатом
  (не нужен домен и отдельный сервис; предупреждение — разовое).
- Токен в URL, не Basic Auth/куки: ссылку из буфера можно просто открыть;
  утечка в историю браузера приемлема для этого сценария.
- Один режим https, не «http для localhost + https наружу»: меньше веток,
  локально ссылка работает так же (принять сертификат один раз).

## Ошибки

- openssl не нашёлся / сертификат не сгенерился → кнопка не включается,
  текст ошибки у 🌐 (существующий `webmic_error`).
- Порт занят → как раньше, ошибка у кнопки.
- Запрос без токена или с чужим токеном → `403`; лимит тела 2 МБ остаётся.
- ipify недоступен (нет интернета/таймаут) → в ссылке LAN-IP; хуже, но
  работоспособно в локалке.

## Тесты

- Юнит: извлечение `t` из URL и проверка токена (валидный/чужой/отсутствует),
  формат новой ссылки-подсказки, существующие тесты `rate_from_url`/PCM.
- E2E: сервер поднят → `curl -k` статики с токеном (200) и без (403);
  POST речевого куска с токеном → finals в ответе; страница с второго
  устройства через публичный IP.
